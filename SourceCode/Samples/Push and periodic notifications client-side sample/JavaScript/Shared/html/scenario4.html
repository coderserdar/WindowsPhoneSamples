In this scenario, the app will update its tile using periodic updates which will poll a web service to retrieve tile XML content.

Periodic notifications are another way of updating tiles. Your app can specify a URI that points to a resource on a web service that contains a tile update notification. The resource should respond to an HTTP GET request and return properly formatted tile or badge notification XML. Your app should call the polling APIs each time your app is launched. A schedule will be set up to continue polling the specified URI on a regular interval. Once you have called the API, Windows will poll this URI regularly and update the associated tiles. Note that your app's package will need to set the "Internet (client)" capability in its manifest.

It is also possible to poll several different URIs simultaneously for tile XML. The tile will automatically cycle through each of the updates. Be sure to enable the notification queue on the tile updater object before calling the periodic update API.

Enter URLs to poll, choose a frequency, and press "Start periodic updates" to begin polling for tile updates.
